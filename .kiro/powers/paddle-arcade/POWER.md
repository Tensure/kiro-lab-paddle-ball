# Paddle Arcade Development Power

This local power packages the repo's game-development guidance as an installable Kiro power example.

Use it when working on:

- Pong solo or local versus gameplay
- Breakout gameplay
- Phaser 3 scenes and systems
- React game shell, menus, settings, and overlays
- synthesized sound effects
- mode-aware powerups
- gameplay tests and property tests
- Kiro specs, steering, hooks, agents, and skills

## Onboarding

1. Read the workspace steering files in `.kiro/steering/`.
2. Read the current baseline spec in `.kiro/specs/paddle-ball-game/`.
3. Confirm whether the project has been scaffolded as Vite React TypeScript yet.
4. If not scaffolded, plan scaffold tasks before implementing Phaser scenes.
5. Use the included steering in `steering/workflow.md` for power-specific workflow reminders.

## Operating Principles

- Specs describe what to build.
- Steering describes durable project rules.
- Skills describe repeatable team workflows.
- Hooks automate recurring review or maintenance actions.
- Agents provide focused personas for architecture, implementation, QA, and Kiro artifact maintenance.
