# Spec-First Game Development

Use this skill when a request needs planning before implementation.

Workflow:

1. Read `.kiro/steering/product-direction.md`, `.kiro/steering/architecture.md`, and `.kiro/steering/game-rules.md`.
2. Check whether the request changes durable product behavior.
3. Update requirements before design, and design before tasks.
4. Keep requirements user-facing and testable.
5. Write acceptance criteria in EARS notation: `WHEN`, `IF`, `WHILE`, `WHERE`, or ubiquitous `THE [system] SHALL`.
6. Keep design focused on architecture, scene/system ownership, data contracts, and correctness properties.
7. Keep tasks small enough for agentic implementation.
8. Link tasks back to requirements.

Use `references/spec-checklist.md` for the acceptance checklist.
